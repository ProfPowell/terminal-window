API Reference

A vanilla JavaScript web component that simulates a terminal console with customizable commands, themes, cursor styles, and typing effects.

Attributes

Configure the terminal using HTML attributes. All attributes can also be set via JavaScript properties.

Appearance

Attribute Description Default
theme Color theme: "dark" or "light" dark
cursor-style Cursor shape: "block", "underline", or "bar" block
cursor-blink Enable cursor blinking true
font-family Font family for terminal text 'Consolas', 'Monaco', monospace
font-size Font size (e.g., "14px", "1rem") 14px
line-height Line height multiplier 1.4

Header

Attribute Description Default
title Title displayed in the header bar Terminal
show-header Show the terminal header bar true
show-controls Show window control buttons (close, minimize, maximize) true
show-copy Show the copy button in the header true
show-theme-toggle Show the theme toggle button true

Behavior

Attribute Description Default
prompt The command prompt text "$ "
typing-effect Enable typewriter animation for output false
typing-speed Milliseconds per character when typing effect is enabled 30
readonly Disable user input (presentation mode) false
max-lines Maximum number of output lines to keep in buffer 1000
welcome Welcome message displayed when terminal loads —
enable-vfs Enable virtual file system with built-in commands (ls, cd, pwd, mkdir, touch, rm, cat) false
persist-history Persist command history to localStorage across sessions false
force-animations Force typing animations even when user has prefers-reduced-motion set false

Events

Listen for these custom events to respond to terminal activity.

command

command

Fired when a command is executed. Detail: { command, args, input }

command-result

command-result

Fired after a command executes successfully. Detail: { command, args, result }

command-error

command-error

Fired when a command fails or is not found. Detail: { command, error }

output

output

Fired when a line is printed to the terminal. Detail: { type, content }

copy

copy

Fired when content is copied to clipboard. Detail: { text, mode }

close

close

Fired when the terminal is closed via the close button

minimize

minimize

Fired when the terminal is minimized/restored. Detail: { minimized }

fullscreen

fullscreen

Fired when fullscreen mode is toggled. Detail: { fullscreen }

interrupt

interrupt

Fired when user presses Ctrl+C

Methods

Control the terminal programmatically using these public methods.

registerCommand()

registerCommand(name, handler)

Register a custom command handler. The handler receives an array of arguments and the terminal instance, and can return a string to print, null to suppress output, or a Promise for async commands.

Parameters:

  • name (string) - Command name (case-insensitive)
  • handler (Function) - Function that receives (args: string[], terminal: TerminalWindow) and returns string|null|Promise
// Simple command terminal.registerCommand(&#39;hello&#39;, () => &#39;Hello, World!&#39;); // Command with arguments terminal.registerCommand(&#39;greet&#39;, (args) => `Hello, ${args[0] || 'stranger'}!`); // Async command terminal.registerCommand(&#39;fetch-data&#39;,

unregisterCommand()

unregisterCommand(name)

Remove a registered command.

Parameters:

  • name (string) - Command name to remove
terminal.unregisterCommand(&#39;greet&#39;);

registerAlias()

registerAlias(alias, command)

Create an alias for a command.

Parameters:

  • alias (string) - Alias name
  • command (string) - Full command string to execute when alias is used
terminal.registerAlias(&#39;ll&#39;, &#39;ls -la&#39;); terminal.registerAlias(&#39;cls&#39;, &#39;clear&#39;);

skipTypingEffect()

skipTypingEffect()

Render output with typing effect

terminal.skipTypingEffect();

clear()

clear()

Clear all terminal output.

terminal.clear();

scrollToBottom()

scrollToBottom()

Force scroll to the bottom of the terminal. The terminal uses smart auto-scroll: it automatically scrolls when new output appears, but stops if the user scrolls up to read previous content. This method forces a scroll and re-enables auto-scroll.

terminal.scrollToBottom();

setInputMask()

setInputMask(masked)

Enable or disable input masking for password prompts. When masked, input is displayed as asterisks (***) instead of the actual characters.

Parameters:

  • masked (boolean) - Whether to mask input
terminal.setInputMask(true); // Shows *** instead of text terminal.setInputMask(false); // Shows normal text

toggleTheme()

toggleTheme()

Toggle between light and dark theme. Announces the theme change to screen readers and fires no events (use attribute observation for reactivity).

terminal.toggleTheme(); // Switches from dark to light or vice versa

close()

close()

Close the terminal by hiding it. Sets `display: none` on the element and dispatches a `close` event. The terminal can be shown again by setting `display: block` or removing the style.

terminal.close(); // Listen for close event terminal.addEventListener(&#39;close&#39;, () => { console.log(&#39;Terminal was closed&#39;); });

minimize()

minimize()

Toggle the minimized state of the terminal. When minimized, only the header is visible. Dispatches a `minimize` event with the new state.

terminal.minimize(); // Minimize the terminal terminal.minimize(); // Restore the terminal // Listen for minimize event terminal.addEventListener(&#39;minimize&#39;, (e) => { console.log(&#39;Minimized:&#39;, e.detail.minimized); });

toggleFullscreen()

toggleFullscreen()

Toggle fullscreen mode (maximize). When fullscreen, the terminal fills the entire viewport. Press Escape to exit. Dispatches a `fullscreen` event with the new state.

terminal.toggleFullscreen(); // Enter fullscreen terminal.toggleFullscreen(); // Exit fullscreen // Listen for fullscreen event terminal.addEventListener(&#39;fullscreen&#39;, (e) => { console.log(&#39;Fullscreen:&#39;, e.detail.fullscreen); });

setTheme()

setTheme(theme)

Set the color theme programmatically.

Parameters:

  • theme (('dark'|'light')) - The theme to apply
terminal.setTheme(&#39;light&#39;); terminal.setTheme(&#39;dark&#39;);

setPrompt()

setPrompt(prompt)

Set the command prompt text programmatically.

Parameters:

  • prompt (string) - The prompt text to display (e.g., "$ ", "> ", "user@host:~$ ")
terminal.setPrompt(&#39;> &#39;); terminal.setPrompt(&#39;user@localhost:~$ &#39;);

setCursorStyle()

setCursorStyle(style)

Set the cursor style programmatically.

Parameters:

  • style (('block'|'underline'|'bar')) - The cursor style to apply
terminal.setCursorStyle(&#39;block&#39;); // █ (default) terminal.setCursorStyle(&#39;underline&#39;); // _ terminal.setCursorStyle(&#39;bar&#39;); // |

setCursorBlink()

setCursorBlink(blink)

Enable or disable cursor blinking programmatically.

Parameters:

  • blink (boolean) - Whether the cursor should blink
terminal.setCursorBlink(true); // Enable blinking terminal.setCursorBlink(false); // Disable

setTypingEffect()

setTypingEffect(enabled, speed=30)

Enable or disable the typing effect for output. When enabled, printed text appears character by character like a typewriter. Respects `prefers-reduced-motion` unless `force-animations` is set.

Parameters:

  • enabled (boolean) - Whether to enable the typing effect
  • speed=30 (number) - Milliseconds per character (lower = faster)
terminal.setTypingEffect(true); // Enable with default

setReadonly()

setReadonly(readonly)

Enable or disable readonly mode. When readonly, the input line is hidden and users cannot type commands. Useful for presentation or demonstration purposes.

Parameters:

  • readonly (boolean) - Whether to enable readonly mode
terminal.setReadonly(true); // Disable input terminal.setReadonly(false); // Enable input

setForceAnimations()

setForceAnimations(force)

Override the user's `prefers-reduced-motion` system preference. When forced, typing animations will run even if the user has reduced motion enabled. Use sparingly and only for testing or when the animation is essential to the experience.

Parameters:

  • force (boolean) - If true, animations will run regardless of system preference
terminal.setForceAnimations(true); // Force animations on terminal.setForceAnimations(false); // Respect system preference

hasReducedMotion()

hasReducedMotion()

Check if the user has enabled `prefers-reduced-motion` in their system settings. When true, the terminal will skip typing animations unless `force-animations` is set.

Returns: boolean - True if the user prefers reduced motion

setI18n()

setI18n(strings, strings.copy, strings.close, strings.minimize, strings.maximize, strings.toggleTheme, strings.copyAll, strings.copyCommandsOnly, strings.copyOutputOnly, strings.copied, strings.commandNotFound)

Set internationalization (i18n) strings for localization. Merges the provided strings with the existing defaults. Triggers a re-render to apply the new strings.

Parameters:

  • strings (Object) - Object containing string keys to override
  • strings.copy (string) - "Copy" button text
  • strings.close (string) - "Close" button tooltip
  • strings.minimize (string) - "Minimize" button tooltip
  • strings.maximize (string) - "Maximize" button tooltip
  • strings.toggleTheme (string) - "Toggle theme" button tooltip
  • strings.copyAll (string) - "Copy All" menu item
  • strings.copyCommandsOnly (string) - "Copy Commands Only" menu item
  • strings.copyOutputOnly (string) - "Copy Output Only" menu item
  • strings.copied (string) - "Copied!" feedback message
  • strings.commandNotFound (string) - "Command not found" error message
terminal.setI18n({ copy: &#39;Copiar&#39;, copied: &#39;¡Copiado!&#39;, commandNotFound: &#39;Comando no encontrado&#39; });

getI18n()

getI18n()

Get a copy of the current i18n strings configuration.

Returns: Object - Current i18n configuration object

const strings = terminal.getI18n(); console.log(strings.copy); // "Copy"

getHistory()

getHistory()

Get the command history array. Returns a copy of the history, not a reference to the internal array.

Returns: string[] - Array of previously executed commands (newest last)

const history = terminal.getHistory(); console.log(history); // ['ls', 'cd /home', 'cat file.txt']

setHistory()

setHistory(history)

Replace the command history with a new array. Useful for restoring history from storage or initializing with predefined commands.

Parameters:

  • history (string[]) - Array of commands to set as history
terminal.setHistory([&#39;ls&#39;, &#39;pwd&#39;, &#39;whoami&#39;]);

clearHistory()

clearHistory()

Clear all command history. If `persist-history` is enabled, also clears the localStorage entry.

terminal.clearHistory();

getContent()

getContent()

Get all terminal output as plain text. Includes both commands (with prompts) and output lines.

Returns: string - All terminal content as newline-separated text

const content = terminal.getContent(); console.log(content); // $ ls // file1.txt file2.txt // $ pwd // /home/user

focus()

focus()

Focus the terminal's input field. Useful after programmatically interacting with the terminal.

terminal.focus();

Slots

Customize content using named slots.

Slot Name Description
title Custom content for the terminal title in the header
actions Custom buttons/actions before the theme and copy buttons
before-output Content inserted before the output area in the terminal body
<terminal-window> <span slot="title">My Custom Terminal</span> <button slot="actions" onclick="runScript()">Run</button> <div slot="before-output"> <p>Welcome to the interactive tutorial!</p> </div> </terminal-window>

Built-in Commands

These commands are available by default:

  • help - Display available commands
  • clear - Clear the terminal output
  • echo [text] - Print text to the terminal
  • history - Show command history

VFS Commands

When enable-vfs="true" is set, these additional commands are available:

  • ls [path] - List directory contents
  • cd [path] - Change current directory
  • pwd - Print working directory
  • mkdir [name] - Create a directory
  • touch [name] - Create an empty file
  • rm [name] - Remove a file or directory
  • cat [file] - Display file contents
