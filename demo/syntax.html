Terminal Window - Syntax & API Reference

A vanilla JavaScript web component that simulates a terminal console with customizable commands, themes, cursor styles, and typing effects.

Installation

Include the script in your HTML file:

<script src="path/to/terminal-window.js"></script>

Or as an ES module:

<script type="module"> import './path/to/terminal-window.js'; </script>

Basic Usage

Add the <terminal-window> element to your HTML:

<terminal-window id="terminal" theme="dark" prompt="$ " title="My Terminal" welcome="Welcome! Type 'help' for commands." ></terminal-window>

Then interact with it via JavaScript:

const terminal = document.getElementById('terminal'); // Print output terminal.print('Hello, World!'); // Execute a command terminal.executeCommand('help'); // Register a custom command terminal.registerCommand('greet', (args) => { return `Hello, ${args[0] || 'stranger'}!`; });

JavaScript Methods

Output Methods

print(text, type = 'output')

Print text to the terminal. The type parameter controls styling:

  • 'output' - Default text color
  • 'error' - Red error text
  • 'info' - Cyan info text
  • 'success' - Green success text
terminal.print('Operation complete!', 'success'); terminal.print('Something went wrong', 'error'); terminal.print('Processing...', 'info');

clear()

Clear all terminal output.

terminal.clear();

Command Methods

executeCommand(input, addToHistory = true)

Execute a command string programmatically.

terminal.executeCommand('ls -la'); terminal.executeCommand('help', false); // Don't add to history

executeSequence(commands, defaultDelay = 1000)

Execute multiple commands with delays (useful for demos).

// Simple array of command strings await terminal.executeSequence(['pwd', 'ls', 'whoami'], 1000); // Or with custom delays per command await terminal.executeSequence([ { command: 'cd /home', delay: 500 }, { command: 'ls -la', delay: 1500 }, 'pwd' // Uses default delay ], 1000);

registerCommand(name, handler)

Register a custom command. The handler receives an array of arguments and the terminal instance.

terminal.registerCommand('greet', (args, term) => { const name = args[0] || 'World'; return `Hello, ${name}!`; }); // Async commands are supported terminal.registerCommand('fetch-data', async (args) => { const response = await fetch('/api/data'); const data = await response.json(); return JSON.stringify(data, null, 2); }); // Return null to suppress output terminal.registerCommand('silent', () => { // Do something silently return null; });

unregisterCommand(name)

Remove a registered command.

terminal.unregisterCommand('greet');

registerAlias(alias, command)

Create an alias for a command.

terminal.registerAlias('ll', 'ls -la'); terminal.registerAlias('cls', 'clear');

Configuration Methods

setTheme(theme)

Set the color theme programmatically.

terminal.setTheme('light'); terminal.setTheme('dark');

toggleTheme()

Toggle between light and dark themes.

terminal.toggleTheme();

setPrompt(prompt)

Change the command prompt.

terminal.setPrompt('root# '); terminal.setPrompt('>>> ');

setCursorStyle(style)

Set cursor style: 'block', 'underline', or 'bar'.

terminal.setCursorStyle('bar');

setCursorBlink(enabled)

Enable or disable cursor blinking.

terminal.setCursorBlink(false);

setTypingEffect(enabled, speed = 30)

Enable/disable typing animation and set speed. The typing effect includes these UX improvements:

  • Reduced motion - Automatically disabled when user has prefers-reduced-motion set
  • Cancellable - Press Ctrl+C or click the terminal to skip to the end
  • Buffered output - Multiple print calls queue up and execute sequentially
terminal.setTypingEffect(true, 50); // 50ms per character

skipTypingEffect()

Skip the current typing animation and show output immediately.

terminal.skipTypingEffect();

setReadonly(readonly)

Enable or disable read-only mode (hides input line).

terminal.setReadonly(true); // Presentation mode

setInputMask(masked)

Mask input characters (for password prompts).

terminal.setInputMask(true); // Shows *** instead of text

Window Control Methods

close()

Hide the terminal (sets display: none).

minimize()

Toggle minimized state (collapses terminal body).

toggleFullscreen()

Toggle fullscreen mode. Press Escape to exit.

Utility Methods

focus()

Focus the terminal input.

getContent()

Get all terminal output as a string.

const content = terminal.getContent(); console.log(content);

copyContent(mode = 'all')

Copy content to clipboard. Modes: 'all', 'commands', 'output', 'selection'.

terminal.copyContent('commands'); // Copy only commands entered

History Methods

getHistory()

Get the command history as an array. Useful for persisting history across sessions.

const history = terminal.getHistory(); localStorage.setItem('terminalHistory', JSON.stringify(history));

setHistory(history)

Set the command history from an array. Useful for restoring history from storage.

const saved = localStorage.getItem('terminalHistory'); if (saved) { terminal.setHistory(JSON.parse(saved)); }

Scroll Methods

scrollToBottom()

Force scroll to the bottom of the terminal. The terminal uses smart auto-scroll: it automatically scrolls when new output appears, but stops if the user scrolls up to read previous content. This method forces a scroll and re-enables auto-scroll.

terminal.scrollToBottom();

Events

The terminal dispatches custom events you can listen for:

command

Fired when a command is executed.

terminal.addEventListener('command', (e) => { console.log('Command:', e.detail.command); console.log('Arguments:', e.detail.args); console.log('Full input:', e.detail.input); });

copy

Fired when content is copied to clipboard.

terminal.addEventListener('copy', (e) => { console.log('Copied text:', e.detail.text); console.log('Copy mode:', e.detail.mode); });

close

Fired when the terminal is closed via the close button.

terminal.addEventListener('close', () => { console.log('Terminal closed'); });

minimize

Fired when the terminal is minimized/restored.

terminal.addEventListener('minimize', (e) => { console.log('Minimized:', e.detail.minimized); });

fullscreen

Fired when fullscreen mode is toggled.

terminal.addEventListener('fullscreen', (e) => { console.log('Fullscreen:', e.detail.fullscreen); });

interrupt

Fired when user presses Ctrl+C.

terminal.addEventListener('interrupt', () => { console.log('User interrupted'); });

command-result

Fired after a command executes successfully.

terminal.addEventListener('command-result', (e) => { console.log('Command:', e.detail.command); console.log('Arguments:', e.detail.args); console.log('Result:', e.detail.result); });

command-error

Fired when a command fails or is not found.

terminal.addEventListener('command-error', (e) => { console.log('Command:', e.detail.command); console.log('Error:', e.detail.error.message); });

output

Fired when a line is printed to the terminal.

terminal.addEventListener('output', (e) => { console.log('Type:', e.detail.type); // 'output', 'error', 'info', 'success' console.log('Content:', e.detail.content); });

These commands are available by default:

Registering Custom Commands

Create interactive terminal experiences by registering your own commands:

// Simple command terminal.registerCommand('hello', () => 'Hello, World!'); // Command with arguments terminal.registerCommand('add', (args) => { const sum = args.reduce((a, b) => a + parseFloat(b), 0); return `Sum: ${sum}`; }); // Async command terminal.registerCommand('weather', async (args) => { const city = args[0] || 'London'; const response = await fetch(`/api/weather?city=${city}`); const data = await response.json(); return `Weather in ${city}: ${data.temp}`; }); // Command that uses the terminal instance terminal.registerCommand('countdown', async (args, term) => { const count = parseInt(args[0]) || 3; for (let i = count; i > 0; i--) { await term.print(`${i}...`); await new Promise(r => setTimeout(r, 1000)); } return 'Blast off!'; }); // Command that changes terminal state terminal.registerCommand('password', (args, term) => { term.setInputMask(true); term.print('Enter password (masked input):'); return null; });

Accessibility

The terminal includes built-in accessibility features:

  • ARIA roles and labels - Proper semantic structure for screen readers
  • Live regions - Output is announced to screen readers as it appears
  • Keyboard navigation - Full keyboard support including history navigation
  • Focus indicators - Visible focus states for keyboard users
  • Reduced motion - Respects prefers-reduced-motion setting
  • High contrast - Supports prefers-contrast: high media query
